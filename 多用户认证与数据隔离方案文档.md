# 多用户认证与数据隔离方案文档

> **版本**：v1.0 | **日期**：2026-06-16 | **状态**：已实施

---

## 1. 概述

知识库系统 V2.0 引入完整的多用户认证体系，支持 Web API 用户和飞书用户统一管理。核心设计原则：

- **数据完全隔离**：不同用户的 Neo4j 图谱数据互不可见
- **统一身份认证**：JWT Token + 飞书 open_id 双通道，可绑定同一账号
- **速率限制**：按用户+操作类型独立限流，防止单用户耗尽系统资源
- **数据可迁移**：飞书自动创建的账号可绑定到 Web 账号，数据自动迁移

## 2. JWT 认证体系

### 2.1 Token 设计

使用 [python-jose](kb/app/auth/jwt.py) 实现 JWT 签发和验证：

| Token 类型 | 有效期 | 配置项 | 用途 |
|-----------|--------|--------|------|
| Access Token | 15 分钟 | `ACCESS_TOKEN_EXPIRE_MINUTES` | API 请求认证 |
| Refresh Token | 7 天 | `REFRESH_TOKEN_EXPIRE_DAYS` | 静默续签 Access Token |

**Token Payload**：
```json
{
    "sub": "usr_2026061612001234567",  // 用户ID
    "username": "alice",                // 用户名
    "exp": 1718536800,                  // 过期时间
    "iat": 1718535900,                  // 签发时间
    "type": "access"                    // token类型
}
```

- 签名算法：HS256（`JWT_ALGORITHM`）
- 签名密钥：`JWT_SECRET_KEY`（生产环境必须设置，建议 ≥32 字符）

### 2.2 密码安全

使用 [bcrypt](kb/app/auth/password.py) 直接哈希（绕过 passlib 兼容性问题）：

```python
def hash_password(plain: str) -> str:
    raw = plain.encode("utf-8")[:72]  # bcrypt 72 字节限制
    return bcrypt.hashpw(raw, bcrypt.gensalt()).decode("utf-8")
```

## 3. 用户存储

### 3.1 SQLite 数据表

[UserStore](kb/app/auth/user_store.py) 使用 aiosqlite 异步 SQLite，三张核心表：

**users** — 用户主表：
```sql
CREATE TABLE users (
    id           TEXT PRIMARY KEY,      -- usr_{timestamp}
    username     TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at   TEXT NOT NULL,
    is_service   INTEGER DEFAULT 0      -- 1=服务账号
);
```

**feishu_user_mappings** — 飞书映射表：
```sql
CREATE TABLE feishu_user_mappings (
    open_id      TEXT PRIMARY KEY,      -- 飞书 open_id
    user_id      TEXT NOT NULL,         -- 关联 users.id
    feishu_name  TEXT DEFAULT '',
    created_at   TEXT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**token_usage_log** — Token 用量日志：
```sql
CREATE TABLE token_usage_log (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id      TEXT NOT NULL,
    action       TEXT NOT NULL,
    tokens_used  INTEGER DEFAULT 0,
    created_at   TEXT NOT NULL
);
```

### 3.2 账号类型

| 类型 | 创建方式 | 示例 | is_service |
|------|----------|------|------------|
| Web 用户 | `/api/auth/register` | `alice` | 0 |
| 飞书自动创建 | 首次消息自动创建 | `feishu_a1b2c3d4e5f6` | 0 |
| 服务账号 | 启动时自动创建 | `default` | 1 |

## 4. 认证中间件

### 4.1 三级依赖注入

[deps.py](kb/app/auth/deps.py) 提供三个 FastAPI 依赖：

**① `get_current_user`** — 严格 JWT 认证：
```python
async def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(_bearer_scheme)) -> CurrentUser:
    # 仅接受 JWT access token，失败返回 401
```

**② `get_current_user_or_service`** — JWT + 服务账号回退：
```python
async def get_current_user_or_service(...) -> CurrentUser:
    # 先尝试 JWT，失败后尝试 knowledge_api_token（MiroMind 等服务调用）
```

**③ `get_current_user_with_rate_limit`** — JWT + 速率限制：
```python
async def get_current_user_with_rate_limit(request: Request, current_user = Depends(get_current_user)) -> CurrentUser:
    # 认证通过后检查速率限制
```

### 4.2 ContextVar 注入

认证成功后，自动将 `user_id` 注入 ContextVar：

```python
Neo4jDatabase.set_current_user(user_id)
```

所有后续数据库查询通过 `execute_read_for_user` / `execute_write_for_user` 自动携带 `$_user_id` 参数，实现透明隔离。

## 5. Neo4j 数据隔离

### 5.1 复合唯一约束

所有用户数据节点使用 `(id, user_id)` 复合唯一约束：

```cypher
CREATE CONSTRAINT entity_id_user IF NOT EXISTS
FOR (n:Entity) REQUIRE (n.id, n.user_id) IS UNIQUE

CREATE CONSTRAINT concept_id_user IF NOT EXISTS
FOR (n:Concept) REQUIRE (n.id, n.user_id) IS UNIQUE
```

### 5.2 用户感知查询

所有图谱查询通过 `execute_read_for_user` / `execute_write_for_user` 自动注入 `$_user_id`：

```cypher
-- 查询时自动隔离
MATCH (n:Entity) WHERE n.user_id = $_user_id ...

-- 写入时自动标记
CREATE (n:Entity {id: $id, user_id: $_user_id, ...})
```

### 5.3 User-ID 索引

为所有节点类型创建 `user_id` 索引确保隔离查询性能：

```
entity_user_id, concept_user_id, source_user_id, cluster_user_id, ingest_record_user_id
```

## 6. 速率限制

### 6.1 Token Bucket 算法

[UserRateLimiter](kb/app/auth/rate_limit.py) 使用令牌桶算法，per-user + per-action 独立限流：

| 操作 | 配额 | 窗口 |
|------|------|------|
| ingest | 10 次 | 60 秒 |
| query | 30 次 | 60 秒 |

### 6.2 工作机制

1. 每个用户+操作组合独立维护一个令牌桶
2. 令牌按 `max_tokens / window` 速率匀速补充
3. 每次请求消耗 1 个令牌，不足时返回 `429 Too Many Requests`
4. 服务账号（`is_service=True`）不受速率限制

## 7. 飞书多用户绑定

### 7.1 命令链

| 命令 | 说明 |
|------|------|
| `/register 用户名 密码` | 从飞书注册新 Web 账号并自动绑定 |
| `/bind 用户名 密码` | 绑定飞书到已有 Web 账号（数据迁移） |
| `/unbind` | 解除飞书绑定 |
| `/whoami` | 查看当前绑定状态 |

### 7.2 数据迁移（两阶段幂等）

当用户将飞书账号绑定到不同 Web 账号时，触发 [_migrate_neo4j_data](kb/app/auth/user_store.py#L317-L384)：

**Phase 1 — 同名合并**：同名节点吸收 aliases/tags/summary 后删除旧节点
**Phase 2 — 直接迁移**：剩余节点 `SET n.user_id = $new_uid`
