## 一、方案概述

### 1.1 设计目标

本方案设计飞书平台作为个人知识库系统的**主要交互入口**，实现：

```
输入：用户通过飞书消息（文本/图片/文件/语音/链接）将知识送入 Ingest 管道
查询：用户通过自然语言在飞书中查询知识库内容
展示：查询结果通过飞书卡片结构化呈现（含关系路径可视化）
反馈：输入处理过程的三级渐进式反馈
体验：拟人化交互，简洁直观
```

### 1.2 设计原则

```
原则 1：飞书只做"采集"和"展示"，不做"编译"
  消息接收 → 构造标准请求 → 调用 Knowledge API
  查询结果 ← 卡片渲染 ← Knowledge API 返回
  所有 LLM 调用发生在后端 Ingest/Query 管道中

原则 2：三层意图识别自动区分输入 vs 查询
  Layer 1：显式前缀（/q、/search、/research 等）→ 0ms
  Layer 2：规则引擎（_QUERY_PATTERNS、_INPUT_PATTERNS 等正则）→ 0ms
  Layer 3：LLM 快速判断（deepseek-v4-flash, 2s 超时）→ ~300ms
  默认行为 = 输入（保存知识）
  自然语言查询 = 自动识别（无需 /q 前缀）

原则 3：异步反馈，绝不沉默
  收到消息 → 1秒内回复确认
  处理完成 → 主动发送结果通知
  处理失败 → 发送错误说明 + 建议

原则 4：拟人化交互
  不是冷冰冰的命令行工具
  用友好、简洁的语言引导用户
  错误时给建议，而非只报错
```

### 3.2 六种消息类型处理

#### (1) 文本消息

```
接收格式（飞书事件）：
  msg_type: "text"
  content: {"text": "RAG 是 Retrieval Augmented Generation 的缩写，它结合了检索和生成..."}

处理流程：
  1. 提取 text 字段
  2. 检测是否包含 URL（http:// 或 https://）
     ├── 含 URL → format="url", source=URL
     └── 不含 → format="text", source=原文
  3. 构造 Ingest 请求

伪代码：
  async def handle_text(content: dict, message_id: str):
      text = content["text"]
      
      # 检测 URL
      if re.match(r'https?://\S+', text.strip()):
          payload = {
              "source": text.strip(),
              "options": {"format": "url", "channel": "feishu"}
          }
      else:
          payload = {
              "source": text,
              "options": {"format": "text", "channel": "feishu"}
          }
      
      # 调用 Ingest API
      result = await ingest_api(payload)
      
      # 发送即时确认卡片
      await send_card(message_id, build_ack_card(result["task_id"]))
```

#### (2) 富文本消息

```
接收格式：
  msg_type: "post"
  content: {
    "title": "文章标题",
    "content": [
      [{"tag": "text", "text": "正文内容"}],
      [{"tag": "a", "href": "https://...", "text": "链接文字"}],
      [{"tag": "img", "image_key": "img_xxx"}],
      [{"tag": "code_block", "language": "PYTHON", "text": "code..."}]
    ]
  }

处理流程：
  1. 遍历 content 数组，按 tag 类型拼接
     ├── text → 直接拼接文本
     ├── a → 拼接为 Markdown 链接 [text](href)
     ├── img → 标记图片位置 [图片]
     ├── code_block → 拼接为 Markdown 代码块
     ├── at → 忽略
     └── hr → 插入分隔线 ---
  2. 如果提取到链接，额外标注 options.url
  3. format="markdown", source=拼接后的内容

伪代码：
  async def handle_rich_text(content: dict, message_id: str):
      parts = []
      urls = []
      for line in content.get("content", []):
          for element in line:
              tag = element.get("tag")
              if tag == "text":
                  parts.append(element["text"])
              elif tag == "a":
                  parts.append(f"[{element['text']}]({element['href']})")
                  urls.append(element["href"])
              elif tag == "code_block":
                  lang = element.get("language", "").lower()
                  parts.append(f"```{lang}\n{element['text']}\n```")
              elif tag == "img":
                  parts.append("[图片]")
              elif tag == "hr":
                  parts.append("---")
      
      markdown_text = "\n".join(parts)
      payload = {
          "source": markdown_text,
          "options": {
              "format": "markdown",
              "channel": "feishu",
              "title": content.get("title", ""),
              "urls": urls
          }
      }
      result = await ingest_api(payload)
      await send_card(message_id, build_ack_card(result["task_id"]))
```

#### (3) 图片消息

```
接收格式：
  msg_type: "image"
  content: {"image_key": "img_4adb3cc3-902b-4187-b0f1-842f67fd017g"}

处理流程：
  1. 通过飞书 API 下载图片
     GET /open-apis/im/v1/messages/{message_id}/resources/{image_key}
     Headers: Authorization: Bearer <tenant_access_token>
  2. 将图片字节 Base64 编码
  3. format="image", source=Base64字符串
  4. 后端 Ingest 管道 Step 1 使用多模态 LLM 或 OCR 提取文字

伪代码：
  async def handle_image(content: dict, message_id: str):
      image_key = content["image_key"]
      
      # 下载图片
      image_bytes = await feishu_client.download_resource(message_id, image_key)
      
      payload = {
          "source": base64.b64encode(image_bytes).decode(),
          "options": {
              "format": "image",
              "channel": "feishu",
              "file_name": f"{image_key}.jpg",
              "file_mime": "image/jpeg"
          }
      }
      result = await ingest_api(payload)
      await send_card(message_id, build_ack_card(result["task_id"]))
```

#### (4) 文件消息

```
接收格式：
  msg_type: "file"
  content: {"file_key": "75235e0c-xxx", "file_name": "论文.pdf"}

处理流程：
  1. 通过飞书 API 下载文件
     GET /open-apis/im/v1/messages/{message_id}/resources/{file_key}
  2. Base64 编码
  3. format="file", source=Base64字符串
  4. 后端 Ingest 管道 Step 1 根据 MIME 类型选择处理方式：
     - PDF → pymupdf 提取
     - Word → python-docx 提取
     - 其他文本 → 直接读取
  5. 文件大小限制：50MB

伪代码：
  async def handle_file(content: dict, message_id: str):
      file_key = content["file_key"]
      file_name = content.get("file_name", "unknown")
      
      # 文件大小检查（飞书文件限制 100MB，我们限制 50MB）
      file_bytes = await feishu_client.download_resource(message_id, file_key)
      if len(file_bytes) > 50 * 1024 * 1024:
          await reply_text(message_id, 
              "📎 文件太大了（超过 50MB），处理起来比较吃力。\n"
              "试试拆分成小一点的文件？或者直接把关键内容复制粘贴给我。")
          return
      
      # 推断 MIME 类型
      mime_type = guess_mime_type(file_name)
      
      payload = {
          "source": base64.b64encode(file_bytes).decode(),
          "options": {
              "format": "file",
              "channel": "feishu",
              "file_name": file_name,
              "file_mime": mime_type
          }
      }
      result = await ingest_api(payload)
      await send_card(message_id, build_ack_card(result["task_id"]))
```

#### (5) 音频消息

```
接收格式：
  msg_type: "audio"
  content: {"file_key": "75235e0c-xxx", "duration": 5000}

处理流程：
  1. 通过飞书 API 下载音频文件
  2. 后端 Ingest 管道 Step 1 使用 DashScope Paraformer 转文字
  3. 转文字后走标准文本处理流程
  4. 附加音频时长信息

伪代码：
  async def handle_audio(content: dict, message_id: str):
      file_key = content["file_key"]
      duration = content.get("duration", 0)
      duration_sec = duration / 1000
      
      # 下载音频
      audio_bytes = await feishu_client.download_resource(message_id, file_key)
      
      payload = {
          "source": base64.b64encode(audio_bytes).decode(),
          "options": {
              "format": "audio",
              "channel": "feishu",
              "file_name": f"voice_{message_id}.ogg",
              "file_mime": "audio/ogg",
              "duration_seconds": duration_sec
          }
      }
      result = await ingest_api(payload)
      
      # 确认卡片中标注音频时长
      card = build_ack_card(result["task_id"], 
                           extra=f"🎵 语音消息 ({duration_sec:.0f}秒)")
      await send_card(message_id, card)
```

#### (6) URL 链接（文本中检测）

```
处理流程：
  1. 在文本消息处理中检测 URL 模式
  2. 如果用户只发了一个 URL（无其他文字）→ format="url"
  3. 如果 URL 嵌在文字中 → format="text"，URL 作为附加信息
  4. 后端 Ingest 管道 Step 1 使用 readability 提取网页正文

判断逻辑：
  text = "https://example.com/article"
  → 仅 URL → format="url"
  
  text = "这篇文章讲的 RAG 技术很好 https://example.com/article"
  → 文本 + URL → format="text", options.urls=["https://..."]
```


## 四、知识查询功能

### 4.1 查询触发机制

```
三层意图识别自动区分输入 vs 查询：

  === 显式前缀（Layer 1, 0ms）===
  /q RAG 和知识图谱有什么关系？     → 查询
  /research RAG 技术趋势            → 深度研究
  /search 知识图谱                  → 关键词搜索
  /recent                           → 最近记录
  /end                              → 结束对话上下文
  /stats                            → 知识统计
  /help                             → 帮助
  /register                         → 用户注册
  /bind                             → 绑定飞书账号
  /unbind                           → 解除绑定
  /whoami                           → 查看当前用户

  === 规则引擎（Layer 2, 0ms）===
  "RAG 和知识图谱有什么关系？"     → 匹配 _QUERY_PATTERNS → 查询
  "帮我研究一下 RAG 的最新进展"    → 匹配 _RESEARCH_PATTERNS → 研究
  "https://xhslink.com/abc123"     → 匹配社交URL正则 → 抓取+输入

  === LLM 快速判断（Layer 3, ~300ms）===
  deepseek-v4-flash JSON 输出：{intent: "query|input|research|social"}
  2s 超时 → 默认 input

  RAG 是 Retrieval Augmented Generation 的缩写...
  
  ↑ 无意图特征 → 默认输入流程
```

### 4.2 查询处理流程

```
用户发送问题（/q 前缀 or 意图识别为 query）
        │
        ▼
  飞书消息适配层
  ├── 提取问题文本（去掉 /q 前缀，或使用原始文本）
  ├── ConversationContext follow-up 富化（指代词替换 + 上下文实体继承）
  ├── 发送“正在思考...”即时反馈
  │
  ▼
  POST /api/query
  Headers: Authorization: Bearer <JWT_ACCESS_TOKEN>
  Body: { "question": "RAG 和知识图谱有什么关系？", "user_id": "xxx" }
        │
        ▼
  后端 Query 管道（V1.2 五步处理）
  │
  ├── Step 1: 查询理解（jieba POS + 规则+LLM 混合分类）
  │     识别实体：[RAG, 知识图谱]
  │     查询类型：关联查询 (relational)
  │     ambiguity 阈值：15
  │
  ├── Step 1.5: 实体解析（EntityResolver 三层匹配）★ V1.2 新增
  │     Layer 1 精确：n.id = $name
  │     Layer 2 CONTAINS：n.id CONTAINS $name
  │     Layer 3 模糊：difflib SequenceMatcher >= 0.6
  │     无匹配 → 生成搜索建议 Top-5
  │
  ├── Step 2: Graph 检索（多用户 user_id 隔离）
  │     4 策略 dispatch：factual / relational / reasoning / global
  │     Cypher: MATCH (n)-[r]->(m) WHERE n.id IN [...] AND n.user_id = $uid
  │     返回：节点内容 + 显式关系 + 隐式关系
  │
  ├── Step 2.5: 质量门判定（QualityGate）★ V1.2 新增
  │     SUFFICIENT → 正常生成
  │     PARTIAL → 降级回答 + 搜索建议
  │     INSUFFICIENT → 拦截，不调用 LLM
  │     Factual 快速路径：单节点 → 直接返回 content（0 次 LLM）
  │
  └── Step 3: LLM 生成回答（DeepSeek deepseek-chat）
        基于检索结果生成结构化回答
        │
        ▼
  构造查询结果卡片 → 发送到飞书
  PipelineTrace 记录全链路 timing
```

### 4.3 四种查询深度的展示策略

| 查询类型 | 触发信号 | 检索策略 | 飞书展示 |
|---------|---------|---------|---------|
| 事实查询 | "是什么/定义/全称" | 单节点 content | 摘要卡片 |
| 关联查询 | "关系/区别/对比" | 节点 + 一跳关系 | 双节点 + 关系路径 |
| 推理查询 | "为什么/导致/因果" | 多跳路径 + 隐式关系 | 推理链 + 置信度 |
| 全局查询 | "总结/趋势/整体" | 群落扫描 + 桥接 | 群落概览 + 桥接节点 |

### 8.2 认证与安全

```
三层认证：

  1. 飞书 → 服务器（入站安全）
     ──────────────────────────
     方式 A：WebSocket 长连接（推荐）
     - 服务器主动连接飞书 WebSocket 端点
     - 连接时携带 app_id + app_secret 鉴权
     - 无需公网回调地址
     - auto-reconnect + watchdog 心跳检测

     方式 B：Webhook 回调
     - 飞书事件推送携带签名，服务端验证：
     - Verification Token 校验
     - Encrypt Key 解密（可选但推荐）

     实现：
       @router.post("/webhook/feishu")
       async def feishu_webhook(request: Request):
           # 验证签名
           timestamp = request.headers.get("X-Lark-Signature-Timestamp")
           nonce = request.headers.get("X-Lark-Signature-Nonce")
           signature = request.headers.get("X-Lark-Signature")
           
           if not verify_feishu_signature(timestamp, nonce, signature):
               raise HTTPException(403, "签名验证失败")
           
           body = await request.json()
           # ... 处理消息

  2. 飞书用户 → 系统用户（多用户绑定）
     ──────────────────────────────────
     飞书 open_id → SQLite UserStore 查询 → user_id
     未绑定用户 → 拦截所有操作，提示 /register + /bind
     已绑定用户 → JWT token 注入到所有 API 调用
     
     绑定流程：
       /register → 创建用户 → 返回 user_id + api_key
       /bind → 将飞书 open_id 关联到 user_id
       /whoami → 查看当前绑定状态
       /unbind → 解除绑定

  3. 服务器 → Knowledge API（内部安全）
     ──────────────────────────────────
     多用户模式：JWT access token
     Authorization: Bearer <JWT_ACCESS_TOKEN>
     
     Token 包含 user_id，所有 Neo4j 操作通过
     ContextVar user_id + 复合唯一约束隔离
```

```python
class FeishuClient:
    def __init__(self):
        self.app_id = os.environ["FEISHU_APP_ID"]
        self.app_secret = os.environ["FEISHU_APP_SECRET"]
        self._token = None

    async def get_tenant_token(self) -> str:
        """获取飞书 tenant_access_token（自动缓存和刷新）"""
        if self._token and time.time() < self._token_expires:
            return self._token
        resp = await httpx_client.post(

        )
        data = resp.json()
        self._token = data["tenant_access_token"]
        self._token_expires = time.time() + data.get("expire", 7200) - 300
        return self._token
    
    async def download_resource(self, message_id: str, file_key: str) -> bytes:
        """下载消息中的资源文件（图片/文件/音频）"""
        token = await self.get_tenant_token()

            f"/resources/{file_key}",

            params={"type": "file"}

        return resp.content

            },

        )

        """更新已发送的飞书卡片"""

@router.post("/webhook/feishu")
async def feishu_webhook(request: Request, bg: BackgroundTasks):
    body = await request.json()
    
    # URL 验证
    if body.get("type") == "url_verification":
        return {"challenge": body["challenge"]}
    
    # 解析事件
    event = body.get("event", {})
    message = event.get("message", {})
    msg_type = message.get("msg_type", "")
    content = json.loads(message.get("content", "{}"))
    message_id = message.get("message_id", "")
    open_id = event.get("sender", {}).get("sender_id", {}).get("open_id", "")
    
    # 多用户绑定拦截
    user = await user_store.get_by_open_id(open_id)
    if not user:
        bg.add_task(handle_unbinded, message_id)
        return {"code": 0}
    
    # 三层意图识别
    intent = await intent_detector.detect(message, user.id)
    
    # 速率限制
    if not rate_limiter.check(user.id, intent):
        bg.add_task(handle_rate_limited, message_id)
        return {"code": 0}
    
    # 分发
    if intent == "query":
        question = intent_detector.extract_query_text(content, msg_type)
        bg.add_task(handle_query, question, message_id, user)
    elif intent == "research":
        topic = intent_detector.extract_research_topic(content)
        bg.add_task(handle_research, topic, message_id, user)
    elif intent == "social":
        bg.add_task(handle_social_ingest, content, message_id, user)
    elif msg_type == "text":
        text = content.get("text", "").strip()
        if text.startswith("/"):
            bg.add_task(handle_command, text, message_id, user)
        else:
            bg.add_task(handle_ingest_text, content, message_id, user)
    elif msg_type in ("post", "image", "file", "audio"):
        bg.add_task(handle_ingest_media, msg_type, content, message_id, user)
    
    return {"code": 0}

async def handle_ingest_text(content: dict, message_id: str):
    """处理文本输入"""
    text = content.get("text", "").strip()
    if not text:
        await feishu.reply_message(message_id, build_error_card(
            "嗯？你好像没发内容哦 😄\n发点什么给我吧——文字、图片、文件都可以"
        ))
        return
    
    # 发送即时确认卡片
    ack_msg = await feishu.reply_message(message_id, build_ack_card())
    
    # 检测 URL
    if re.match(r'^https?://\S+$', text):
        payload = {"source": text, "options": {"format": "url", "channel": "feishu"}}
    else:
        payload = {"source": text, "options": {"format": "text", "channel": "feishu"}}
    
    # 调用 Ingest API
    result = await call_ingest_api(payload)
    
    # 等待处理完成，发送结果卡片
    final_result = await poll_task_result(result["task_id"])
    
    if final_result["status"] == "completed":
        await feishu.reply_message(message_id, build_complete_card(final_result))
    else:
        await feishu.reply_message(message_id, build_error_card(
            f"处理遇到了一些问题...\n{final_result.get('error', '未知错误')}"
        ))
```

### 8.4 异步任务管理

```
流程：
  1. 收到飞书事件 → 立即返回 {"code": 0}
  2. 将消息处理放入 BackgroundTasks（FastAPI 后台任务）
  3. 后台任务：
     a. 发送即时确认卡片
     b. 调用 Ingest/Query API
     c. 轮询任务状态（GET /api/tasks/{task_id}）
     d. 任务完成后发送结果卡片

轮询策略：
  间隔 1 秒，最多等待 60 秒
  超时后发送"处理超时"通知

备选方案（更高效）：
  Ingest 管道完成后主动回调飞书适配层
  通过 Redis Pub/Sub 或 asyncio.Event 通知
  避免轮询开销
```
