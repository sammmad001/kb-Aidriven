| 痛点 | 你的日常 |

## 五大特色功能

### 1. ⭐ 隐式关系推理 —— 帮你发现「你不知道你知道」的关联

传统笔记软件只能存储你手动标注的链接。本系统在你每次存入新知识时，AI 会自动分析它与已有知识之间的深层关系，发现你从未想到的隐藏关联。

**支持的 9 种关系类型**：

| 关系类型 | 含义 | 示例 |
|---------|------|------|
| 依赖关系 | A 的核心能力依赖 B | RAG 依赖语义理解技术 |
| 取舍关系 | A 和 B 存在矛盾 | 成本优先 vs 质量优先 |
| 桥接关系 | A 连接不同领域 | 成本控制连接技术选型和项目管理 |
| 演进关系 | A 是 B 的前身 | RAG 演进为 GraphRAG |
| 解决关系 | A 解决了 B 的问题 | 知识图谱解决了多跳推理瓶颈 |
| 因果关系 | A 导致 B | 数据爆炸导致传统检索失效 |
| 时序关系 | A 先于 B | 需求分析先于方案设计 |
| 对立关系 | A 与 B 观点矛盾 | 两种架构方案的优劣之争 |
| 类比关系 | A 类似于 B | 知识图谱类似于人脑神经网络 |

每条隐式关系都带有**置信度评分**（0-100%），你可以清楚知道 AI 对这条关联有多大把握。

### 2. 🧩 知识群落自动聚类

系统会自动将相关知识组织成「群落」，类似大脑中自然形成的知识模块。
- 你不需要手动分类或打标签
- 随着知识增长，群落自动演化、合并、分裂
- 一眼看清你的知识结构全貌

### 3. ⚠️ 矛盾检测

当你存入的新知识与已有知识存在冲突时，系统会自动标注：
- **事实冲突** —— 同一件事有两个不同说法
- **观点分歧** —— 两种对立观点共存
- **时效更新** —— 新信息可能需要替换旧信息

你再也不用担心自己收藏了过时的资料却不知道。

### 4. 🔍 质量自检（Lint）

系统每日自动进行知识健康检查：
- 发现「孤儿节点」（没有任何关联的知识）
- 发现重复节点（同一知识存了多份）
- 标记低置信度的关联（需要人工确认）
- 自动清理无效连接

**维护成本**：每月仅需 15 分钟确认关键项。

### 5. 🔬 深度研究

集成专业研究引擎，输入一个主题即可获得深度分析报告：
- 自动检索、综合分析、结构化输出
- 研究结果**自动入库**，与已有知识建立关联
- 形成「研究 → 入库 → 关联 → 再研究」的知识增长闭环

# 第二部分：使用指南

## 快速开始

系统提供两个主要入口：**Web 界面**和**飞书机器人**。你可以任选其一，也可以同时使用。

### 方式一：Web 界面注册登录

1. 浏览器打开系统地址（如 `https://your-domain.com/ui`）
2. 点击「注册」，输入用户名（3-50 个字符，支持字母、数字、下划线）和密码（至少 6 位）
3. 注册成功后自动登录，进入知识图谱主页
4. 左侧导航可切换「图谱浏览」和「智能问答」

> 💡 提示：Web 界面主要用于**浏览知识图谱**和**智能提问**，日常知识录入推荐使用飞书机器人。

### 方式二：飞书机器人注册绑定

如果你主要通过飞书使用，需要先将飞书账号与系统账号绑定：

**第一步：注册系统账号**
```
给机器人发送：/register 你的用户名 你的密码
```
> 用户名要求 3-50 个字符，密码至少 6 位。

**第二步：绑定飞书账号**
```
给机器人发送：/bind 你的用户名 你的密码
```
绑定成功后，机器人会回复确认卡片。此后你在飞书中发送的所有消息都会自动关联到你的知识库。

**查看绑定状态**：发送 `/whoami`
**解除绑定**：发送 `/unbind`

## 知识录入

系统支持 **6 种格式**的知识录入，最常用的方式是通过飞书机器人发消息。

### 飞书端录入（推荐）

| 格式 | 怎么操作 | 适合场景 |
|------|---------|---------|
| 📝 **文本** | 直接发文字消息给机器人 | 随手笔记、灵感、想法 |
| 🖼️ **图片** | 直接发送图片 | 截图、照片、扫描件中的知识 |
| 📄 **文件** | 直接发送文件（PDF、Word 等） | 文档、论文、报告 |
| 🎙️ **音频** | 发送语音消息 | 会议记录、语音备忘 |
| 🔗 **网页链接** | 发送包含链接的消息 | 好文章、参考资料 |
| 📱 **社交链接** | 发送小红书/微博链接 | 社交媒体上的优质内容 |

**操作就这么简单**：选中内容 → 转发给知识库机器人 → 系统自动处理。

**处理反馈**：机器人会通过三级渐进式卡片告知处理进度：
1. **即时确认**（< 1 秒）：显示"已收到，正在处理中..."
2. **分类反馈**（2-5 秒）：显示识别出的类型和关键实体
3. **完成通知**（10-20 秒）：显示创建的节点、发现的关联、耗时统计

### 智能意图识别

你不需要记命令——直接发消息就行。系统会自动判断你是想「存知识」还是「查知识」还是「做研究」：

| 你发送的内容 | 系统判断 | 处理方式 |
|-------------|---------|---------|
| 一段较长的文字 / 链接 / 图片 | 知识录入 | 自动走录入流程 |
| 带问号的问题 / "什么是..." | 知识查询 | 自动走查询流程 |
| "帮我研究..." / "深度调研..." | 深度研究 | 启动研究引擎 |

当然，你也可以用显式命令确保精确控制（见下方）。

### 命令行方式（API）

适合技术用户或自动化场景：

**录入知识**
```bash
curl -X POST https://your-domain.com/api/ingest \
  -H "Authorization: Bearer your-token" \
  -H "Content-Type: application/json" \
  -d '{
    "source": "你的知识内容...",
    "options": {
      "format": "text",
      "tags": ["标签1", "标签2"]
    }
  }'
```

**支持的格式参数**：`text`（纯文本）、`markdown`、`url`（网页链接）、`file`（文件 Base64）、`image`（图片 Base64）、`audio`（音频 Base64）

## 知识查询

### 飞书端查询

**方式一：显式命令**
```
/q 你的问题
```
或
```
/ask 你的问题
```

**方式二：直接提问**（系统自动识别）

直接发送问题消息，系统会自动判断这是一个查询请求：
```
RAG 和 GraphRAG 有什么关系？
```

**方式三：关键词搜索**
```
/search 关键词
```

**对话上下文支持**：

系统支持连续追问，无需每次重复完整问题：
```
你：什么是 RAG？
系统：RAG 是检索增强生成技术...

你：它有什么局限性？
系统：RAG 的主要局限包括多跳推理能力不足...

你：有什么替代方案？
系统：针对 RAG 的局限，主要替代方案有 GraphRAG...
```

> 💡 对话上下文保留最近 5 轮，10 分钟内有效。发送 `/end` 可结束当前对话。

### Web 界面查询

1. 打开 Web 界面，进入「智能问答」页面（`/ui/qa`）
2. 在输入框中输入你的问题
3. 系统返回答案，附带来源引用和置信度

### API 查询

```bash
curl -X POST https://your-domain.com/api/query \
  -H "Authorization: Bearer your-token" \
  -H "Content-Type: application/json" \
  -d '{"question": "你的问题"}'
```

**回答包含**：
- 📝 结构化答案文本
- 📎 引用的知识来源（可追溯）
- 📊 置信度评分
- 🔗 使用的隐式关系（如有）

研究结果卡片包含：
- 📋 结构化研究报告
- 🔗 与已有知识的关联分析
- 📊 关键发现和结论

> 💡 研究结果会自动作为新知识入库，后续查询时可以检索到。

# 第三部分：常见问题

### Q1：支持哪些格式的知识录入？

**6 种格式**：纯文本、Markdown、网页链接、文件（PDF/Word/文本等）、图片（自动 OCR 识别文字）、音频（自动转文字）。

### Q2：我的数据安全吗？

系统采用多用户数据隔离机制，每个用户的知识完全独立。密码采用行业标准加密存储（bcrypt），通信使用 JWT 令牌认证。所有数据存储在私有服务器上，不会被第三方访问。

### Q3：月成本大概多少？

约 **$5-6/月**（约 ¥35-45），主要覆盖 AI 模型调用费用。基础设施（数据库、服务器框架等）均为免费开源软件。

### Q4：AI 自动发现的关联准确吗？

每条 AI 发现的隐式关系都带有**置信度评分**（0-100%）。系统会自动审查低置信度的关联，你也可以通过质量报告手动确认。随着知识积累，AI 的判断会越来越精准。

### Q5：知识录入后多久可以查到？

**1-3 秒**即可查到基础信息（快速路径），完整的 AI 增强（内容编写 + 隐式关系推理）约需 10-20 秒。

### Q6：可以同时从多个渠道使用吗？

可以。飞书、Web 界面、API 命令行可以随意混用，所有数据统一存储在同一个知识图谱中。

### Q7：支持团队协作吗？

当前版本聚焦个人知识管理。多用户已实现数据隔离（每人独立的知识空间），但尚未实现团队共享和协作。这是未来版本的规划方向。

### Q8：已有大量笔记，能批量导入吗？

支持通过 API 批量导入。可以将笔记逐条发送到 `/api/ingest` 接口，系统会自动处理格式转换、去重和关联。

### Q9：可以导出知识吗？

知识以 Markdown 文件形式持续渲染输出到 `wiki/` 目录，可以用 Obsidian 浏览或通过 Git 同步到本地。未来版本将支持 JSON-LD、RDF 等标准格式导出。

### Q10：怎么查看系统运行状态？

访问 `https://your-domain.com/health` 可以查看各组件（数据库、AI 模型、飞书连接）的实时状态。
