| 推荐方案 | 3-7s | ✅ 优秀（< 10s） |

### 5.3 高负载成本变化

- **并发请求**：百炼按请求计费，并发不改变单价
- **限流**：免费账户有 QPS 限制（通常 2-5 QPS），超出会排队
- **Batch 调用**：支持的模型可用 Batch API 半价，但延迟增加到分钟级
  - 适合非实时的批量 Ingest 场景
  - qwen3.5-plus Batch: 输入 ¥0.4 + 输出 ¥2.4 = 节省 50%

## 六、成本优化策略

### 6.1 短期优化（立即可用）

1. **混合模型策略**：不同环节用不同模型（推荐方案已内置）
2. **非思考模式**：Step 2/3d 使用非思考模式，减少思维链 Token 消耗
3. **控制输入长度**：精简 Neo4j 上下文摘要，避免输入膨胀

### 6.2 中期优化（知识积累后）

1. **上下文缓存**：qwen3.7-plus/flash 支持上下文缓存，输入 Token 享折扣
2. **Batch API**：非实时场景（如批量导入历史笔记）使用 Batch 半价
3. **本地 Ollama 替代 Step 2**：当本地 GPU 可用时，Step 2 分类用 Ollama（¥0）

### 6.3 长期优化

1. **模型蒸馏**：积累足够标注数据后，微调小模型替代通用模型
2. **缓存重复查询**：相同查询直接返回缓存结果

## 七、项目配置

### 7.1 .env 配置（推荐方案）

```env
# LLM Provider
LLM_PROVIDER=dashscope
DASHSCOPE_API_KEY=<your-dashscope-api-key>

# 环节模型分配（推荐方案）
DASHSCOPE_MODEL_ANALYZE=qwen-turbo        # Step 2 分析分类
DASHSCOPE_MODEL_COMPILE=qwen3.5-plus      # Step 3b 页面编写
DASHSCOPE_MODEL_REASONING=qwen-flash      # Step 3d 隐式推理
DASHSCOPE_MODEL_QUERY=qwen3.5-plus        # Query 回答生成

# 兼容旧配置（默认模型）
DASHSCOPE_MODEL=qwen3.5-plus
```
